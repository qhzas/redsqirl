HDFS view

Now we have Hadoop file system view open. We will check where our data is stored and go there.

Hover over the Spark Aggregator action and hit Ctrl. You will notice the path line starting with /user. In the Hadoop file sytem, go to the path mentioned either by typing it or clicking.

Once in the folder you will notice there are a lots of files. If you click on them, they will likely be empty. Spark saves the data this way. As Red Sqirl user session is single threaded, it takes a while to go through all those empty files. A simple solution to that problem, would be to use a Spark Data Format action after our aggregator and set the number of partitions to two so it will create two files.
