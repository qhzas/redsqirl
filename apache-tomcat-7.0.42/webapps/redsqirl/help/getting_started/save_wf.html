Save workflow

It is always important to save your work. Although Red Sqirl creates backup in the redsqirl-backup directory, it should not be relied upon too often. Red Sqirl supports *.rs and *.srs files that are saved on Hadoop file system. By default a user saves their workflow under its own file system folder in redsqirl-save, locked from other users. However a workflow can be saved in a shared folder.

  1. In the top menu, go to File > Save.
  2. Name the workflow "getting_started".
  3. Click OK
