Synchronous Source

Overview

A synchronous dataset contains a timestamp within its path to identify its creation date. Synchronous Source is a node used to retrieve data, from a synchronous data set, that will be used in a moving window scheduled process. Red Sqirl supports two types of data storage for synchronous data-set: HCatalog and HDFS.

  • Inputs - No Inputs
  • DataType - Select the Data Storage (HDFS, HCatalog...)
  • SubType - Select the Data Format (Text Format, Compressed Format...)
  • Browser - Select a path, Change the Header
  • Template: define a template path
  • Occurrence: Define the frequency of the data
  • Outputs - The template path as well as the real path of the initial dataset

Source Data Type

Storage

In Storage, the user can select where the data that will be used is stored. Only HCatalog and HDFS data type are supported.

Source Data Sub-Type

Format

The data sub-type defines the format of the user’s data.

Source: Browser

Dataset

Within the dataset browser the user can select the path of the data that will be used.

Data format parameters

The user can change the delimiter. The delimiter can be a normal character (",","|",";") or an ASCII character in which case (#1, #44).

Data preview (HDFS Only)

The user can change the header and fields of a file by clicking on the pencil Icon. A header can be a list of feature names or a list of feature name with type. For example, if I have a data set that contains three fields. I could defined the header to be "field1, field2, field3" or "field1 DATE, field2 STRING, field3 INT".

Template

path

The user can define here what are the parts of the user’s path that make up the timestamp. The user will be able to use the following variables:

  • ${YEAR}
  • ${MONTH}
  • ${DAY}
  • ${HOUR}
  • ${MINUTE}

Example:
The original path may read: /twitter/tweets/dt=2017-10-14;hr=23
The revised template path should be changed to:
/twitter/tweets/dt=${YEAR}-${MONTH}-${DAY};hr=${HOUR}

Path Selection

In here the user will be able to define the frequency of the user's dataset, and the one the user needs for the user's analyses.

Unit

The time unit used for the following fields.

Time Unit

The amount of units between every run.

Example:
If the unit selected is HOUR, a 1 in this field would mean to run every hour; whereas a 2 in this field would mean to run every 2 hours.

Number of instances

The number of the most recent captures to be used in the user’s end analysis.

Example:
If you put in 24 number of instances with a frequency of an hour then you are asking the system to use 24 hours worth of data in the following actions.

Offset

Offset is used when the incoming data is being processed at a time different from when it is received.

Example:
Your data from 8 am is set to process at 9 am. In order for the system to understand this lag you would set the offset to -1 (if your units are in hours this means 1 hour earlier).

Scheduling

Please note that Synchronous Source runs default on the timestamp defined by the data set. E.g if the dataset is daily timestamped,the sync will run everyday and pull in the most recent data set each day. In order to change the frequency of the sync and also define a particular time for the sync to occur please update the scheduling.
