Synchronous Sink

Overview

Synchronous Sink is an action that registers a synchronous data set at the end of a scheduled job. A synchronous dataset contains a timestamp within its path to identify its creation date. Typically a synchronous dataset does not change for a given period of time after its creation. Red Sqirl supports two data storage for synchronous data-set: HCatalog and HDFS. In order to reuse the Sink dataset into another coordinator, you need to link it with a Synchronous Sink Filter.

  • Inputs - Input to register
  • Template path - The path in which the data is stored
  • Outputs - The registered output

Sink

Template Path

The path with at least one of the following time variables. The variable chosen by the user should correspond to the timestamp required by the user.

  • ${YEAR}
  • ${MONTH}
  • ${DAY}
  • ${HOUR}
  • ${MINUTE}

Example:
The original path may read: /twitter/tweets/dt=2017-10-14;hr=23
The revised template path should be changed to:
/twitter/tweets/dt=${YEAR}-${MONTH}-${DAY};hr=${HOUR}
